Welcome to the Boldemort test file.
This document exercises every supported inline style.

--- Bold ---

Use **double asterisks** to make text bold.
You can also use __double underscores__ for the same effect.
Numbers go bold too: **2025** and **42**.
Mixed content: **Hello World 123**.

--- Italic ---

Use *single asterisks* to italicize.
Or use _single underscores_ for the same result.
A sentence: The *quick brown fox* jumps over the lazy dog.
With underscores: The _quick brown fox_ jumps over the lazy dog.

--- Monospace ---

Inline code uses `backticks`.
A command example: run `ls -la` to list files.
Variable names: set `DEBUG=true` before starting.
Digits in mono: `v1.2.3` is the latest release.

--- Strikethrough ---

Use ~~double tildes~~ to strike through text.
This price was ~~100 EUR~~ now only 49 EUR.
Mistakes happen: I thought ~~the meeting was Monday~~ it is Tuesday.

--- Mixed styles in one line ---

**Bold**, *italic*, `mono`, and ~~struck~~ all on one line.
You can have **bold words** next to *italic ones* freely.
Code snippet `print("hello")` alongside **important** notes.

--- Passthrough (must not be altered) ---

Plain text with no markers stays exactly as-is.
Prices like $100 and formulas like x*y are not touched.
A single *dangling asterisk with no close stays raw.
Same for a **dangling double with no close.
An _underscore with no close is also left alone.
